Question: Q: newly created/modified .docx files are not indexed by spotlight

A couple of days ago, I accidentally noticed that spotlight/Finder search cannot find some .docx files which obviously exist in my computer. As mentioned by an old post (Re: Old problem with Spotlight and Word .docx files):

The symptoms of the problem are that Spotlight will not find content in some .docx files. It is not clear why it fails on some .docx files but not on others. This problem does not seem to affect other Microsoft formats such as .doc, .xls, .xlsx, etc.


To further describe the symptoms:

MS .docx files will be correctly indexed and find by Finder search after the mdimport command which forces the Finder to index a particular file:

mdimport filename.docx


However, immediately after a .docx file is opened, it will disappear from the Finder search results. Running the above command again will bring back the docx file to the search results. Big problem!!! I also confirmed that this problem doesn't seem to exist for .doc files.

Therefore the problem is clearly related to the way how .docx files are indexed.


... ... ... ... ... ... ... ...


As mentioned by the Helpful Replies from the same post Re: Old problem with Spotlight and Word .docx files, we can check which module is used to index a particular file by :

mdimport -d1 filename.docx


Strangely enough, It turns out that .doc files are indexed by Microsoft Office.mdimporter located in /Library/Spotlight, whereas .docx files are indexed by RichText.mdimporter located in /Library/Spotlight/System/Library/Spotlight.


So, looks like the Microsoft Office.mdimporter is doing a good job while RichText.mdimporter screws it up. Judged by the module name, I think that the Microsoft Office.mdimporter is the intended (official) way to index MS Office files, therefore should also take care of the .docx files. Then the question is: how can I make .docx files be indexed by Microsoft Office.mdimporter instead of RichText.mdimporter, just as for .doc files.


Here is the gold:

Inside /System/Library/Spotlight/RichText.mdimporter/Contents/Info.plist (this is the plist file that tells RichText.mdimporter what files to be indexed), remove line 24 (remember to keep a backup)

<string>org.openxmlformats.wordprocessingml.document</string>

(.docx file is recongnized as type "org.openxmlformats.wordprocessingml.document", if you run mdimport -d1 to a .docx file you will learn that), so that RichText.mdimporter will no longer handle .docx file .


But to modify this Info.plist file, you will need to disable the System Integrity Protection of macOS first, by

  1. Click the Apple menu.
  2. Select Restart...
  3. Hold down Command + R to boot into Recovery mode.
  4. Click the Utilities menu and select Terminal.
  5. Type 
    csrutil disable
    and press return.
  6. Close the Terminal app.
  7. Click the Apple menu and select Restart...

You should now be able to modify the file, after you can repeat these steps with 

csrutil enable
to secure your machine again.

(quoted from macos - How do I edit an Info.plist file? - Ask Different)


... ... ... ... ... ... ... ...


After I have successfully modified the Info.plist file and restart the Mac, I confirmed that the .docx files are now being indexed by Microsoft Office.mdimporter. (Although there are errors but these errors also appear when a .doc file is being indexed, before I did any modification. So I think it's safe to neglected them.) What is important is that the bottom line tells us the .docx file is now being indexed by Microsoft Office.mdimporter.

Now the .docx files will not disappear from the Finder search results after being opened/modified anymore. So far so good. I will update if anything new appears.
